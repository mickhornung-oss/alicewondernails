# Accessibility Plan

## Ziel

Barrierefreiheit von Anfang an bedenken.

## Fokus für Arbeitsblock 01

- Strukturierte Dokumentation
- Keine UI-Neuerstellungen im Backend-Fundament
- Keine neuen fehlerhaften HTML-Templates

## Spätere Anforderungen

- semantische HTML-Struktur
- Tastatur-Navigation
- Kontrastanforderungen
- Screenreader-Kompatibilität
- Formular- und Fehlermeldungen
